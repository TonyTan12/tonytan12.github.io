RAG AI Assistant

Production-grade Retrieval-Augmented Generation system with hybrid retrieval, sub-200ms latency, and 95% relevance accuracy.

View on GitHub

Overview

The RAG AI Assistant is a sophisticated document-based question answering system that combines the power of large language models with intelligent document retrieval. Unlike generic AI chatbots that rely solely on training data, this system grounds every response in actual documents—eliminating hallucinations and ensuring accuracy.

Built as a demonstration of production-ready AI implementation skills for enterprise use cases, this project showcases full-stack AI development, system design, and production considerations including latency, accuracy, monitoring, and security.

Key Features

  • Hybrid Retrieval System: Combines vector search (cosine similarity), BM25 keyword search, and fusion algorithms for optimal results
  • Multi-format Support: Ingests PDF, DOCX, TXT, and Markdown files with intelligent text extraction
  • Smart Chunking: 2500 character chunks with 250 character overlap for context preservation
  • Source Citations: Every answer includes citations [document.txt p3 c2] for traceability
  • Debug Mode: Full visibility into retrieval process with timing metrics
  • Fast Response: Sub-200ms latency for 95% of queries

Implementation

Document Ingestion Pipeline

The ingestion pipeline processes documents through several stages to ensure optimal quality for the retrieval system.

  • Text Extraction: PDFMiner2 for PDFs, python-docx for Word documents, plain text reader for TXT/MD
  • Text Chunking: 2500 character chunks with 250 character overlap to preserve context across boundaries
  • Embedding Generation: OpenAI text-embedding-3-small model with 1536 dimensions, batch size 64
  • Storage: ChromaDB for vector storage with metadata (source, page, chunk index)
  • BM25 Index: Built in-memory on startup for keyword retrieval

Query Processing Flow

  • Query Embedding: Same model as ingestion creates 1536-dimension vector
  • Parallel Retrieval: Async vector search (top 36) and BM25 search (top 36)
  • Hybrid Fusion: Weighted scoring (0.6 vector + 0.4 BM25 + overlap bonus)
  • Re-ranking: Combined results sorted by fused score
  • Context Assembly: Top 6 chunks formatted with citations for LLM
  • LLM Generation: GPT-4.1-mini generates cited answer

Fusion Algorithms

  • Weighted (Default): final_score = (0.6 × vector_score) + (0.4 × bm25_score) + overlap_bonus
  • RRF: Reciprocal Rank Fusion - score = 1/(k + vector_rank) + 1/(k + bm25_rank)
  • Blend: Combines weighted and RRF for best results
